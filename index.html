Elite Dangerous — Controller Binding Layouts

CMDR John DeParagon — WinWing HOTAS & VirPil HOSAS

Two independent presets sharing only the RC1-Falcon pedals. Playing in VR (Quest 3). On-foot is mouse + keyboard and is intentionally excluded.

Design rules applied

  • Axis-first. Anything that can be an axis and has a free axis is one (thrusters, throttle, yaw, sensor zoom / FSS tuning, TDC/FSS camera).
  • Blind-reachable for VR. UI nav, panel focus, and panel-tab cycling live on hats/knobs so you never touch the keyboard in the headset.
  • One control per binding within a category. No ED function is mapped to two controls in the same category. Different categories (Flight, SRV, FSS, DSS, Galaxy Map, Fighter, Camera Suite, Free Camera) may reuse the same physical control — only one category is ever active at a time, so there's no conflict. This is how the camera and scanner modes get rich control without consuming flight buttons.
  • Headlook stays unbound. The headset handles look; recenter is the Meta button via the VR runtime (plus an ED Reset HMD bind, below).

Hardware constraints captured (read before binding)

  • No throttle detent. Throttle is forward-only; reverse is a Toggle Reverse Throttle button (grip thumb #1 aft). Set the SimApp afterburner ratio so the AB detent doesn't consume throttle range.
  • Finger lifts are not buttons — physical travel stops to reach hard-0 and hard-max. Don't bind them; the AB zone is not Boost.
  • TDC / Castle slew is a 2-axis (analog) — thrust in flight, camera in FSS/DSS/Free Camera.
  • Left-grip wheel = Sensor Zoom (flight) / FSS Tuning (FSS) — single axis, two category binds.
  • Slide and Dial are axes — both currently spare (sensor zoom/tuning moved to the wheel).
  • Gray knobs (HDG, CRS, PANEL LTS, HMD) each give three inputs: momentary CCW, momentary CW, push.
  • 2-way switches rest down = not engaged; a momentary flick up is the click.
  • 3-way switches rest center; each side is a momentary press on entry.
  • Reset pips = balance — one binding. Master Mode is not an ED concept; A/G and A/A are ordinary buttons. Orbit lines never bound.
  • Panel Lights = ship/SRV lights.
  • Charge ECM is the real action (no "Field Neutraliser" exists).

VR notes (Quest 3)

  • Recenter view: Meta button on the headset, or the in-game Reset HMD Orientation (Config A pinky-toggle down; Config B left Hat A push).
  • HeadLook: every HeadLook bind empty — headset supersedes.
  • Panels / UI / maps: on momentary hats/knobs for blind reach.
  • Larger HUD text recommended in VR.

JoyToKey — optional

Most switches bind directly (press fires on the flick; return to rest). JoyToKey only needed to hold a sustained state from a switch position — enable the per-button "press once even if held" option, then bind that key in ED. Your OS combos (CTRL-V, SHIFT-WIN-D) can live on spare switches via JoyToKey.

Bind in-game so ED writes the correct device IDs, then export your .binds and I can sanity-check the device strings against this layout.
