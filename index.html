Elite Dangerous — Controller Binding Layouts

CMDR John DeParagon — WinWing HOTAS & VirPil HOSAS

Two independent presets sharing only the RC1-Falcon pedals. Playing in VR (Quest 3). On-foot is mouse + keyboard and is intentionally excluded.

Design rules applied

  • Axis-first. Any function that can be an axis and has a free axis is bound to one (thrusters, throttle, yaw).
  • Blind-reachable for VR. No function requires looking at the device. Two things are reserved on every rig: a hat for panel focus, and the thumb ministick for UI navigation inside the focused panel — so you drive menus without touching the keyboard.
  • Click vs. hold doubling. ED fires a different command on a short press vs. a hold of the same button. Where it's sensible, one physical control carries two related functions (e.g. click = Supercruise, hold = Hyperspace jump). These are marked [C] (click) and [H] (hold).
  • Headlook stays unbound. The headset handles look; recenter is the Meta button (hold), not an ED bind.

A note on ED click-vs-hold

In Controls → flight, the same action listing accepts a primary bind; ED distinguishes a tap from a long-press internally for actions that support it (and you can pair two distinct actions on one button only where the game treats one as hold). Where I list [C]/[H] on one control, bind the click action to the momentary, and the hold action to its "(Hold)" or long-press variant. If a pairing doesn't expose a hold variant in-game, split it to two controls — both rigs have spare inputs.

Hardware constraints captured (read before binding)

  • No throttle detent. The Orion 2 throttle has no center detent, so throttle runs in forward-only range. Reverse is reached with a Toggle Reverse Throttle button on the grip (slider lever), not by dropping below a detent.
  • Toe brakes are not usable as a drive/throttle axis. ED reads the RC1-Falcon toe brakes as non-centering inputs — releasing them does not return to neutral, so they can't drive SRV or flight throttle. Pedals are yaw-only; SRV drive moves to the throttle main lever.
  • A/G and A/A are two separate momentary buttons. Their backlight is DCS-only and does nothing in ED — ignore it.
  • Reset pips = balance (a single ED action; do not map two buttons).
  • Gray knobs (PANEL LTS, HDG, CRS, Dial, Slider) each give three inputs: momentary on left-turn, momentary on right-turn, and a push click. Treat each as 3 buttons, not an axis.
  • Maintained switches (WING FOLD, LDG GEAR, PARK BRK, FLAP, HOOK, LAUNCH BAR) hold a position, so ED sees a continuously-held button. Use JoyToKey edge-trigger for a single keystroke per flip, or bind one position only. Three-way switches rest in the middle by default.

On JoyToKey edge-triggering: it can do this, but it's not the default. JoyToKey treats a held switch as a held key by default. To get a single keystroke on the edge, open the button's properties and enable the option to press the key only once even while the button is held (one-shot). That yields one keystroke when the switch flips into that position and nothing while held. The flip back is a separate JoyToKey button (the other position), assigned the same key — so every flip = one keystroke = one ED toggle.

If that's fiddly, Option 2 is bulletproof: bind only one position of each switch directly in ED and never use the other as a bind. You lose the toggle-both-ways elegance but it always works with no middleware. Practical recommendation: Option 2 for switches you rarely flip mid-flight (landing gear, silent running) and JoyToKey only for anything toggled often — which, in ED, is almost nothing, so you may not need JoyToKey at all.

VR notes (Quest 3)

  • Recenter view: hold the Meta button on the headset, or bind it in SteamVR / OpenXR / OpenComposite controller bindings — there is no native ED keybind for HMD recenter, so don't waste an ED slot on it. (The "Recenter VR" entries above assume you route it through your VR runtime, optionally triggered by the listed stick button.)
  • HeadLook: leave every HeadLook bind empty — the headset supersedes it.
  • Galaxy/System map, panels, UI nav: all kept on momentary hats/ministicks so you're never hunting for a key while wearing the headset.
  • GUIs in VR read better with larger HUD text; unrelated to binds but worth setting alongside.

JoyToKey & "always-on" switches (WinWing only)

  • The 3-stage and dual-stage switches maintain state → ED sees a continuously-held button. For momentary ED actions that's wrong, so use JoyToKey to emit a single keystroke on the state edge (enable the "press key only once even if held" one-shot option per button), then bind that key in ED.
  • Three-way switches rest in the middle by default; you flip to a position only when using that function.
  • Good candidates to live on maintained switches via JoyToKey: landing gear, cargo scoop, orbit lines, silent running, night vision, lights, master mode (combat/explore), plus your existing OS combos (CTRL-V, SHIFT-WIN-D).
  • Reserve 2–3 momentary press switches for true direct-to-ED binds (heat sink / chaff / shield cell) where edge-vs-hold timing matters.
  • Solo player extras worth a switch if you want them: mute/PTT (for the two EU squadmates), screenshot, EDMC/overlay toggle.
