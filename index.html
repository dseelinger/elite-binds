Elite Dangerous — Controller Binding Layouts

CMDR John DeParagon — WinWing HOTAS & VirPil HOSAS

Two independent presets sharing only the RC1-Falcon pedals. Playing in VR (Quest 3). On-foot is mouse + keyboard and is intentionally excluded. Ease ratings (1 = easiest reach, 10 = finger gymnastics) shown per control on Config A.

Design rules applied

  • Easiest controls carry the highest-frequency actions. Placement follows the ease ratings, and a 3-way (2 useful positions) beats a 4-way hat (4 positions) for quick blind use.
  • Every hat is device-prefixed — "Stick Coolie" vs "Throttle Coolie" — so no row is ambiguous on its own. Both the stick and the throttle grip have a Coolie hat and a black button hat.
  • One control per binding within a category. Different categories (Flight, SRV, FSS, DSS, Galaxy Map, Fighter, Camera Suite, Free Camera) reuse the same controls — only one is active at a time.
  • Axis-first for anything analog. Headlook unbound (headset handles look).

Hardware constraints captured (read before binding)

  • NEVER bind trigger stage 2. On this stick a stage-2 pull also fires the stage-1 weapon unintentionally — so stage 2 is left completely unbound, and Secondary fire lives on the front-top Red Button instead.
  • No throttle detent. Throttle is forward-only; reverse = Toggle Reverse Throttle on Throttle thumb #1 aft. Set the SimApp afterburner ratio so the AB detent doesn't eat throttle range.
  • Finger lifts are not buttons (physical travel stops). The AB zone is not Boost.
  • Throttle slew is a 2-axis (analog) — thrust in flight, camera in FSS/DSS/Free Camera.
  • Left-grip wheel ("Dial") = Sensor Zoom (flight) / FSS Tuning (FSS) — one axis, two category binds.
  • Gray knobs (HDG, CRS, PANEL LTS, HMD) each give three inputs: CCW, CW, push.
  • 2-way switches rest down; flick up = the click. 3-way switches rest center; each side is a momentary press.
  • Reset pips = balance (one binding). Master Mode isn't an ED concept; A/G and A/A are ordinary buttons. Orbit lines never bound. Panel Lights = ship/SRV lights. Charge ECM is the real action (no "Field Neutraliser").

JoyToKey — optional

Most switches bind directly (press fires on the flick; return to rest). JoyToKey only needed to hold a sustained state — enable the per-button "press once even if held" option, then bind that key in ED.
